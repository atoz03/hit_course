# 学术写作简要复习大纲

- 研究生定位与培养：科研能力与素质、类型差异（应用/学术、本科/研究生、硕/博）、导师角色、基本要求与心态。
- 选题策略：课题/论题/题目区分，选题原则（方向/兴趣/资料/能力/范围/创新），确定流程与开题前准备。
- 研究方法：问题发现、idea获取与实现、验证与可复现、个人管理与常见误区。
- 文献检索与综述：渠道与数据库、检索技巧、筛选标准、阅读与笔记、梳理写法与管理工具。
- 开题/中检/答辩：开题报告要点、中期检查、答辩与PPT建议。
- 学位论文撰写：结构要素、摘要与关键词、章节格式、公式图表、实验与结论、工程应用型补充。
- 学术论文写作：研究/综述论文结构与写作范式，信息密度与简洁。
- 学术规范与伦理：不端类型、引文与署名致谢规范、规章与案例、诚信倡议。

# 学术写作与学术规范复习笔记

## 第一讲 如何选题和做研究

### 第一部分：什么是研究生？

#### 1. 总览

* **定义**：研究生就是做研究的学生。
* **基本要求**：必须满足培养方案中的所有要求（课程、学分、考试、论文）。
* **核心观点**：论文是“干”出来的，不是埋头“写”出来的，更不是“抄”来的。

#### 2. 研究生培养的目的

* **科研能力和科研素质**：
  * 科研创新能力。
  * 发现和解决问题的能力。
  * 资料收集和处理的能力。
  * 逻辑思维、语言表达和文字写作能力。
  * 快速动手实现的能力。
* **科研意识与精神**：
  * 想不想做研究（意识）。
  * 适不适合做研究（素质）。
  * 能不能做研究（方法）。
* **分类**：
  * 应用型：侧重工程或管理实践能力。
  * 学术型：重点培养从事科学研究工作的能力。

#### 3. 如何具备科研能力

如果目前不具备，需要：

* **科研方法的磨练**：综合利用旧知识，学习新知识，阅读文献。
* **科研实践的磨练**：参与导师项目，尽快进入实验环节（先知其然，后知其所以然）。

#### 4. 导师的作用

* 不再教更多具体知识，而是教“如何自己学习知识”。
* **重点培养三种能力**：创新能力、动手能力、终身学习能力。
* **师生关系**：

  * **兔子理论**（华罗庚）：导师指出兔子在哪里，教你打兔子的本领（经验）。
  * 运动员与教练员的关系。
  * 同事关系（导师 Know something, not everything）。
  * **需要避免的误区：**

    - 并非所有兴趣都该纵容：莫把“兴趣”当作挑肥拣瘦的借口：必要时要忍痛割爱
    - 请不要拿这些让你的导师“理屈词穷”：研究生做项目是在给导师打工，导师在论文中的署名不合理，我的论文创新性已经足以毕业

#### 5. 研究生的基本要求

- 事实证明

  - 能力弱和态度差，常常相伴而生
- 研究生的基本要求

  - **“小事愿干，中事能干，大事敢干”**
- “尽力而为”vs“竭尽全力”

  - 若干个A相乘，A*A*A…=0，“尽力而为”者用99.99%（＝A）的力气去做事
  - 若干个B 相乘，B*B*B…=1，“竭尽全力”者用100.01%（=B）的力气去做事

#### 6. 本科教育 vs 研究生教育

* **本科**：通识教育+素质教育，以学习为主（输入、接受、记忆）。
* **研究生**：专业教育，以创新为主（输出、创新、创造）。
  * **硕士**：培养工作技能，以实践为主，注重普遍性研究技能。
  * **博士**：培养学术技能，理论+实践，强调批判精神和发现问题的能力。
  * **专业水平的差别，不是素质的差别**
* 教育的共性：提高 **研究能力 = 观察能力 + 分析能力**
  - 观察能力：别人观察不到，而你能观察到的能力
  - 分析能力：别人想不到，而你能想到的能力

#### 7.科研的历程和境界

- 学习为主，要摸到科技的前沿
- 研究为主，要占据科技的前沿
- 猜想为主，要引领科技的前沿

#### 8.具体的科研工作

- 吸收性工作：读文献，避免低水平重复
- 创造性工作：从无到有提出模型、方法和方案，反复论证和实验
- 实现性工作：及时总结提炼，撰写并发表论文

### 第二部分：如何选题？

#### 1. 何为选题？

选择论文的论题

- 选择确定所要研究论述的问题和内容
- 论题选得好——事半功倍
- 选题正确，具有重要的理论意义和实用价值

三个名词：

- 课题：某一学科重大的科研项目，它的研究范围比论题大得多
- 选题：是指选择论文的论题
- 题目：论文的标题，它的研究范围一般比论题要小

#### 2. 选题的重要性

* **战术角度**：决定论文价值，反映学术水平，规划工作方向，提高研究能力。
* **战略角度**：面向国家重大需求，责任担当意识（“国家需要就是我的专业”），重能力，而轻知识的年代，敢为人先，不怕挑战。

#### 3. 选题基本原则

1. **结合专业方向**：源于项目高于项目，从实践/文献中发现。
2. **结合个人兴趣**：扬长避短，兴趣可以培养。
3. **考虑资料储备**：数据、资料、设备、经费（切忌进入空白领域）。
4. **结合个人能力**：量力而行（基础好选难/复杂，基础弱选小）。
5. **范围适中**：大题小做，小题大做。
6. **切入口要小**：便于入手，避免太宽泛（如“xx关键技术研究”太宽泛）。
7. **要新更要实**：避免盲目随大流，慎选有争议题目。
   * **三点支撑**：热点 (Hot topic)、前沿 (难点)、经典 (学术/应用价值)。

### 第三部分：如何做研究？

#### 1. 何为研究？

研究≠研发：研究是发现新知识发明新技术，研发是新产品新系统。

#### 2. 研究过程

Topic (选题) -> Problem (发现问题) -> Idea (分析问题) -> Concrete work (解决问题) -> Paper writing -> Submit

#### 3. 发现问题

* **科学研究的心脏**：提出问题比解决问题更重要。
* **方法**：读文献，读文献，读文献。
  * 浮在表面 -> 问题少，易重复。
  * 深入下去 -> 问题空间大，易创新。

#### 4. 寻找 Idea

- 不要指望导师给出 idea

* **途径**：

  * 读经典、最新、顶刊顶会文献（学术只有“奥运会”）。
  * 了解哪些解决了，哪些没解决，为什么？
  * 从实验的意外发现中选题。
* **读文献技巧**：

  - 大量阅读文献:外文文献、中文文献、学位论文

  * 先泛读（摘要、引言、结论），后精读。
  * 手脑并用：边抄边看（抄基本思想、公式推导）。
  * 亲自动手复现，积累直觉。
* **提升理解**：分类 -> 分析（逻辑思维） -> 归纳 -> 综合（发散思维）。

#### 5. 实现与验证 Idea

* **实现**：理论分析（无漏洞）、实验验证（周全）、数据（可复现）、比较（不可少）。
* **验证**：动手实验。数据可靠，结果准确。
  * 一半时间做实验，一半时间看文献（追踪动态）。
  * 交流是最好的老师。

#### 6. 治学境界 (王国维 vs 研究生)

1. 被导师 Push。
2. 无需导师 Push。
3. 主动 Push 导师。

## 第二讲：如何检索文献

### 1. 为什么要检索

* **目的**：了解动态、掌握技术、避免重复、判断价值、寻找创新。
* **现状**：不检索 = 闭门造车。Internet 提供了与世界同步的机会。

### 2. 常用数据库

* **校内资源**：工大图书馆。
* **中文**：中国知网 (CNKI)、万方、超星。
* **CS关键全文库**：
  * ACM (美国计算机学会)
  * IEEE/IEE Electronic Library
  * Elsevier Science (ScienceDirect)
  * Springer, Wiley 等。
* **索引数据库**：
  * **Ei 工程索引**。
  * **SCI 科学引文索引** (信息全，检索功能强)。
* **工具**：百度学术、Google Scholar (Glgoo)、CCF 推荐列表、LetPub (查IF)、ESI。

### 3. 检索技巧

* **流程**：分析课题提取概念 -> 扩展概念 (同义词) -> 构建策略 (AND/OR) -> 调整策略。
* **布尔逻辑**：
  * **OR**：扩大范围 (A OR B)。
  * **AND**：缩小范围 (A AND B)。
  * **NOT**：排除 (NOT C)。
* **高级技巧**：
  * **精确查找**：用引号，如 `"vulnerability detection"`。
  * **通配符**：`*` (0-n个字符), `?` (1个字符)。
  * **优先顺序**：使用括号 `()` 改变优先级。
  * **连字号**：`deep-learning-based`。

### 4. 文献筛选与调研

* **筛选标准**：高被引、高等级期刊/会议、牛人/团队、时间新（近3年）。
* **调研步骤**：
  1. 读书打基础。
  2. 找**综述**（中文综述水平不错，博士论文综述好）。
  3. 看**会议**（最新成果）。
  4. 研读**期刊**（权威）。
  5. 利用**参考文献**顺藤摸瓜。
* **方法**：
  * **浏览捕捉法**：广泛浏览，寻找问题，不能先入为主。
  * **追溯验证法**：初步确定选题后，先有拟想，后查证。
* **好习惯：**随时记录，及时回溯，及时结合，收集完整
* **技巧：**多数文章看摘要，少数文章看全文，综述要看related，集中时间看文献，做好记录，要引用的多看，注意参考价值高低

### 5. 如何阅读文献

- **阅读量要求**：硕士 60-80 篇，博士 120-180 篇。
- **阅读过程**：阅读中文综述-英文综述-泛读经典精读-最新的英文论文（顶会顶刊）
- **避免忘记**：及时整理，随时记录，反复阅读
- **问题意识**：基于大量分析复现，批判形式思维（批判性看文献，正确看待负面结果），不要一味追求完美

### 6. 文献综述与管理

* **梳理**：不是罗列，要兼顾综合性和代表性。
* **结构**：学派发展式、年代序列式、问题导向式（推荐）。
* **管理工具**：EndNote, NoteExpress 等。

## 第三讲：开题和中检

### 1. 开题报告

* **目的**：明确内容、方案、条件、进度，保证质量。
* **核心内容**：
  * 为什么做？（背景、意义）。
  * 做什么？（研究内容）。
  * 怎么做？（研究方案）。
* **考察点：**选题正确性，研究内容恰当性，前期准备工作充分性，研究方案合理性
* **主要内容：**课题意义-文献综述-（前期成果）-主要内容-进度安排-（创新点）-经费/困难/参考文献
* **区别**：
  * **学术型**：突出研究色彩，理论/算法创新，“面向/基于...的研究”。
  * **专业型**：突出工程色彩，系统/算法优化，“小题大做”。
* **常见问题**：
  * 题目过大/过小，核心概念过多（建议不超过2个）。
  * 内容太散（拼盘式），缺乏逻辑体系。
  * 系统开发类缺少需求分析和关键技术。
  * 现状分析流水账，缺少深入评价。
  * 进度安排过于粗略。
  * 参考文献陈旧或数量不足。（硕 20 外文 1/3 5k 字 博 50 外文 1/2 1.5w 字）

### 2. 中期检查

* **目的**：过程管理，及时纠偏。
* **考察点**：按计划进行？完成80%？有阶段性成果（初步实验/代码）？
* **报告撰写**：
  * 给出详细设计、核心算法流程、接口说明。
  * 给出关键测试数据。
  * 图文并茂。

### 3. 答辩 PPT 制作

* **原则**：内容简洁清晰，图文并茂，重点讲原理思路，非单纯念稿。
* **配色**：
  * 深底浅字或浅底深字（对比度高）。
  * 暖色兴奋，冷色冷静。
  * 利用互补色强调。
* **结构**：背景意义一带而过，重点在工作量和创新点。

## 第四讲：学位论文撰写

### 1.类型：

- 基础理论型：需要提出成体系的理论
- 工程

### 1. 写作步骤

数据采集 -> 实验研究 -> 整理分析 -> 拟定提纲 -> 完成初稿 -> 补充修改 -> 定稿。

### 2. 论文构成

题目、摘要、目录、正文（绪论、主体、结论）、致谢、参考文献、附录。

### 3. 各部分撰写规范

#### 题目

* 恰当准确，中文不超过25字，不设副标题。

#### 摘要 (Abstract)

* **四要素**：
  1. **背景/目的**：为什么做？
  2. **内容/方法**：做了什么？怎么做的？
  3. **结果**：得到了什么数据/效果？
  4. **结论**：有什么价值/意义？
* **要求**：独立自含，第三人称，不引用文献，不列公式图表。
* **常见问题**：背景写太多，工作写太少；流水账；评价主观。

#### 关键词

* 5-6个，从大到小排列，分号隔开。

#### 章节结构

* 硕士 4-5 章，博士 5 章左右。
* 标题不加标点，不使用非通用缩写。

#### 绪论 (Introduction)

* 背景意义、国内外现状（分类综述，忌流水账）、研究内容、章节安排。
* **引用规范**：按顺序编码 [1]，上标。

#### 正文 (Body)

* **逻辑**：层次感、缜密性、科学性。
* **图表公式规范**：
  * **公式**：居中，编号 (1-1) 右对齐。变量首次出现要解释。
  * **插图**：图题在下 (图1-1)，具有自明性。流程图要规范（单入口单出口）。
  * **表格**：表题在上 (表1-1)，三线表。
  * **引用**：先见文字，后见图表。

#### 实验结果与分析

* 数据真实，环境可复现。
* **必须有分析**：不能只罗列结果，要解释原因（Why strong/weak?）。
* 比较时使用统计检验。

#### 结论 (Conclusion)

* **内容**：重申重要性 -> 总结主要成果及创新点 (1,2,3点) -> 不足 -> 展望。
* **区别**：摘要是给没看全文的人看，结论是给看完的人看。
* **禁忌**：不重复摘要，不出现公式图表，不自我评价（如“首次提出”）。

#### 参考文献

* **数量**：硕士 >40篇 (外文>20)，博士 >100篇。
* **时效**：近5年不少于1/3。
* **格式**：严格遵守 GB/T 7713 标准（[M]专著, [J]期刊, [C]会议, [D]学位论文）。

## 第五部分：学术论文写作 (Paper)

### 1. 特点

* **强套路性**：约定俗成的结构。
* **冷静文风**：Fact-based，客观。
* **强逻辑性**：剥洋葱式展示，严谨（不把话说满）。
* **创新性**：New (新领域/方法) 或 Better (改进)。

### 2. 结构 (Hourglass Structure - 漏斗型)

Title -> Abstract -> Introduction (宽) -> Related Work -> Method -> Experiments (窄) -> Discussion -> Conclusion (宽)

### 3. 各部分写作要点

#### 标题 (Title)

* 准确、简短（短标题往往引用更高）、便于检索。
* 避免大而空（如“...的研究”），避免非通用缩写。

#### 摘要 (Abstract)

* **中文四要素**：目的、方法、结果、结论。
* **英文五要素**：Context, Objective, Method, Results, Conclusion。
* **Tips**：不要分段，不引文献。

#### 引言 (Introduction)

* **范式**：背景 -> 前人工作 (Known) -> 遗留问题 (Unknown/Gap) -> 本文贡献 (Contribution) -> 结构安排。
* **原则**：开门见山，突出重要性，不贬低别人，不夸大自己。

#### 相关工作 (Related Work)

* **梳理主线**：按方法、时间或问题分类。
* **核心**：你做的与前人有什么不同？

#### 本文方法 (Our Solution)

* **内容**：Motivation, Definition, Theory, Algorithm, Diagram。
* **原则**：读者思维（Reader-friendly），符号自明，逻辑连贯。

#### 实验与讨论 (Experiments & Discussion)

* **实验**：设计周全，数据公开，可复现。
* **讨论**：
  * 解释结果（因果、对比）。
  * **Threats to Validity**：不回避负面结果和不足。

#### 结论 (Conclusion)

* 总结发现，展望未来。

## 第六部分：学术规范

### 1. 内涵与意义

* **学术规范**：学术共同体共同遵守的准则。
* **意义**：学术生命线，整治生态，增强自律。

### 2. 学术不端行为

* **表现**：
  * **伪造/篡改**：捏造数据，更改记录。
  * **剽窃/抄袭**：窃取他人思想/文字，未注明出处。
  * **一稿多投**。
  * **不当署名**：无实质贡献挂名，或剥夺贡献者署名。
* **抄袭界定**：
  * 引用原话未加引号。
  * 概述观点未加标注。
  * 连续引用超过一定字数（如250字）。
  * **自我抄袭**也算不端。

### 3. 引文规范

* **原则**：以原始文献为主，避免转引（二手文献）。
* **标注**：凡引用他人观点、数据、方案，必须标注。
